Allows you to run a DELETE query against a database using data obtained from previous steps. Runs a query for each element in the stream.

Parameters:

  • dbName String

    A database name that matches a object key defined in your Nextract config file

  • tableName String

    Table name

  • matchCriteria Array

    Array of objects with key/value params to be subbed out in a parameterized query. The expected format is [{ tableColumn: '', comparator: '', collectionField: '' }].

Example:

var matchCriteria = [
                       { tableColumn: 'first_name', comparator: '=', collectionField: 'first_name' },
                       { tableColumn: 'last_name', comparator: '=', collectionField: 'last_name' }
                      ];
return ETL.Plugins.Core.Database.deleteQuery('nextract_sample', 'users', matchCriteria);

Allows you to run a select query against a database using data obtained from previous steps. Runs a query for each row in the stream. Each stream element will be returned with the properties of the original item data + the properties of the joined data. Optionally returns all or just matched rows.

Parameters:

  • dbName String

    A database name that matches a object key defined in your Nextract config file

  • sqlStatment String

    The sql to run for each row in the collection. The WHERE clause should contain some ? placeholder references to match on against the current row. The query must return only 1 matching row.

  • joinFilterColumns Array

    An array of properties to use as replacements for the ? placeholders. Pulls the values from the incoming element.

  • joinColumnsToReturn Array

    A list of the properties to return in the unmatched case. Should normally match the columns defined in the SELECT clause. Only required if returnedUnmatched is set to true.

  • returnedUnmatched Boolean

    (optional: defaults to true) Returns all original collection items will null as the value for properties missed in the join. If true, then joinColumnsToReturn must be given.

Returns:

Stream:

Returns stream element with added properties from the join query

Example:

var joinSQL = 'select first_name, last_name from users where id = ?';
var joinFilterColumns = [ 'id' ];
var joinColumnsToReturn = ['first_name', 'last_name'];
ETL.Plugins.Core.Database.joinQuery(streamElement, 'nextract_sample', joinSQL, joinFilterColumns, joinColumnsToReturn, true);

Defined in build/plugins/core/Database/Database.js:128

Query interface for select statements

Parameters:

  • dbName String

    A database name that matches a object key defined in your Nextract config file

  • sql String

    SQL statement to execute. Can be a fully formed SQL select statement or a parameterized one with ":key" placeholders. If the later, then sqlReplacements must be an object of key/values to be replaced.

  • sqlReplacements Object

    (optional) List of key/value params to be subbed out in a parameterized query

Returns:

Promise:

Returns a promise resolved with a read stream to use in conjuction with pipe()

Example:

var sql = 'select first_name, last_name, age, salary from users where id = :id';
var sqlParams = { id: id };
ETL.Plugins.Core.Database.selectQuery('dbname', sql, sqlParams);
