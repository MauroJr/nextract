runInWorker

(
  • workerMsg
)
Promise

Defined in build/plugins/pluginBase.js:86

Convinence method used by plugins to run code in a background web worker thread. A worker script must exist in the same directory as the plugin itself with a name of Worker.js. The format of the worker script must match the one defined in the Sort plugin for example (see src/plugins/core/Sort/Worker.js);

Parameters:

  • workerMsg Object

    The message to be passed to the worker. The object contains two properties; 1) cmd - The command/method to run inside the Worker & 2) args - The arguments being passed to the methiod defined in command via apply().

Returns:

Promise:

Promise resolved with collection

Example:

var workerMsg = { cmd: 'orderBy', args: [collection, iteratees, orders] };
return pluginUtils.runInWorker(workerMsg);
