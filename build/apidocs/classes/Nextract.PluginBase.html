  • streamFunctionType
)
Function

Defined in build/plugins/pluginBase.js:425

Accepts a stream transform function that conforms to the through2.obj stream wrapper API. Caller can choose to use through2 or one of the through2 helper modules (map, filter, spy). For more information see - https://github.com/rvagg/through2.

Parameters:

  • streamFunction Function

    Function that conforms to the through2.obj stream wrapper API

  • streamFlushFunction Function

    Function that conforms to the through2.obj stream flush method API (pass null for no flush)

  • streamFunctionType String

    (optional, defaults to standard through2) standard, filter, map, reduce, or spy

Returns:

Function:

Returns stream transform wrapped using through2

Example:

var streamFunction = function(element, index) { return element.foo <= 10 };
return pluginName.runTask('sometaskname', streamFunction, 'filter');

(
  • streamFunction
)
Promise

Defined in build/plugins/pluginBase.js:296

Some streams require a more traditional blocking like operation (e.g.) Sort, Group By, etc. We want to process the stream and return back a stream without breaking the stream pipe or giving the appearance of a blocking action. This method will take the first item of a stream and use it to create a temporary table in our internal database by inspecting its keys and values.

Parameters:

  • streamFunction Object

    The first object/element of a stream

Returns:

Promise:

Resolves with the temporary table name

Example:

See the sortBy method of the core Sort plugin

runInternalQuery

(
  • sql
  • sqlReplacements
  • expectsResults
)
Promise

Defined in build/plugins/pluginBase.js:225

Provides access to internal sqlite3 database. Allows plugins to run raw db queries as needed.

Parameters:

  • sql String

    SQL statement to execute. Can be a fully formed SQL select statement or a parameterized one with ":key" placeholders. If the later, then sqlReplacements must be an object of key/values to be replaced.

  • sqlReplacements Object

    (optional) List of key/value params to be subbed out in a parameterized query

  • expectsResults Boolean

    Should this query return results?

Returns:

Promise:

If expectsResults was true then the resolved promise should include an array of query results. Otherwise, resolve is empty.

Example:

var sql = 'select first_name, last_name, age, salary from users where id = :id';
var sqlParams = { id: id };
somePlugin.runInternalQuery(insertSql, sqlReplacements, false);

runInWorker

(
  • workerMsg
)
Promise

Defined in build/plugins/pluginBase.js:377

Convinence method used by plugins to run code in a background web worker thread. A worker script must exist in the same directory as the plugin itself with a name of Worker.js. The format of the worker script must match the one defined by the npm workerjs module - https://www.npmjs.com/package/workerjs#node-mode---allowing-require

Parameters:

  • workerMsg Object

    The message to be passed to the worker (can be an object)

Returns:

Promise:

Promise resolved with worker response msg

Example:

var workerMsg = { cmd: 'orderBy', args: [collection, iteratees, orders] };
return pluginUtils.runInWorker(workerMsg);
